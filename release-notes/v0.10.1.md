Desktop Mac banner correction

## What's new

- **Corrected desktop Mac info banner.** IOKit dumps from @jshier's Mac Studio (M3 Ultra) on [#95](https://github.com/darrylmorley/whatcable/issues/95) confirmed that desktop Macs expose full port data through the same HPM path as laptops. Cable e-markers, device enumeration, and Thunderbolt fabric all work normally. The only thing missing is charger identity (FedDetails) from the battery controller. The GUI and CLI banners now say exactly that instead of the misleading "per-port PD diagnostics are not available."
